List:

Menyimpan dan Mengambil Data: List adalah kumpulan elemen yang disusun dalam urutan tertentu. Untuk menyimpan data dalam daftar, Anda menambahkan elemen ke dalamnya menggunakan indeks. Untuk mengambil data, Anda mengakses elemen dengan indeks yang sesuai.
Karakteristik:
Elemen dalam daftar diindeks, sehingga Anda dapat mengaksesnya dengan cepat.
List mempertahankan urutan elemen.
Kelebihan:
Memungkinkan duplikasi elemen.
Fleksibel, Anda dapat menambahkan, menghapus, atau mengubah elemen dengan mudah.
Kekurangan:
Pencarian elemen memerlukan waktu yang lebih lama untuk daftar besar karena perlu memeriksa setiap elemen.


Set:

Menyimpan dan Mengambil Data: Set adalah kumpulan elemen unik tanpa urutan tertentu. Untuk menyimpan data, Anda menambahkan elemen ke dalam set. Untuk mengambil data, Anda melakukan pencarian elemen dalam set.
Karakteristik:
Set tidak mempertahankan urutan elemen.
Set hanya menyimpan elemen unik, sehingga tidak ada duplikasi.
Kelebihan:
Mengecek keanggotaan (membership) sangat cepat karena set menggunakan struktur data hash.
Ideal untuk menghapus duplikasi atau menghitung nilai unik dalam kumpulan data.
Kekurangan:
Tidak dapat mengakses elemen berdasarkan indeks karena set tidak mempertahankan urutan.


Map:

Menyimpan dan Mengambil Data: Map (peta) adalah kumpulan pasangan kunci-nilai. Untuk menyimpan data, Anda menambahkan pasangan kunci-nilai ke dalam peta. Untuk mengambil data, Anda mencari nilai dengan kunci yang sesuai.
Karakteristik:
Setiap kunci dalam peta adalah unik.
Tidak ada ketergantungan pada urutan untuk mengakses nilai dalam peta.
Kelebihan:
Memungkinkan pengindeksan data berdasarkan kunci, yang memungkinkan akses cepat ke nilai.
Ideal untuk merepresentasikan hubungan antara dua jenis data yang berbeda.
Kekurangan:
Membutuhkan lebih banyak memori karena menyimpan pasangan kunci-nilai.
Operasi pencarian khusus pada nilai membutuhkan lebih banyak waktu karena peta tidak diurutkan berdasarkan nilai.
